# Python Mail Client (Socket Implementation)
Консольный почтовый клиент, реализованный на "голых" сокетах без использования высокоуровневых библиотек (вроде `imaplib` или `smtplib`).
## Основные возможности
### Работа с почтой (IMAP)
* **Низкоуровневая работа через сокеты**: Использование модулей `socket` и `ssl` для установления защищенных соединений.
* **Парсинг BODYSTRUCTURE**: Рекурсивный обход AST-дерева структуры письма для поиска текстовых частей и вложений.
* **Декодирование MIME**: Поддержка форматов `Base64` и `Quoted-Printable`, а также корректная склейка многобайтовых символов UTF-8 в заголовках.
* **Очистка HTML**: Автоматическое извлечение чистого текста из HTML-писем с сохранением форматирования (удаление тегов, скриптов и стилей).
* **Управление ящиком**: Создание папок, перенос и удаление писем, навигация между папками сервера.
* **Работа с вложениями**: Просмотр количества, общего объема и скачивание файлов в исходном бинарном виде.

### Отправка писем (SMTP)
* **Поддержка STARTTLS/SSL**: Безопасная авторизация `AUTH LOGIN`.
* **Отправка вложений**: Сборка `multipart/mixed` сообщений с возможностью прикрепления изображений (автоопределение MIME-типа по сигнатуре файла).

### Надежность
* **Авто-реконнект**: Система автоматического переподключения и авторизации при обрыве связи или таймауте сервера.
* **Безопасность**: Скрытый ввод пароля через `getpass`.

### 🛠 Установка и запуск

1. Клонируйте репозиторий:
   ```
   git clone https://github.com/stepantsib/Mail-Client.git
    ```
2. Запустите приложение:
   ```Bash
   cd mail-client
   python app_mail_client.py
   ```
Требования

    Python 3.13+
    Для работы с Gmail/Yandex/Mail.ru требуется использовать пароль приложения.

Структура проекта

    app_mailClient.py — Главный CLI-интерфейс и логика сессий.
    imap_client.py — Реализация IMAP протокола и парсинга контента.
    smtp_client.py — Реализация SMTP протокола и формирования сообщений.
